## üß† What is a Provisioner?

A **Terraform provisioner** is a way to execute custom commands or scripts on the **local machine** or **remote resource** **after** it's created or before it's destroyed.

> _Provisioners act as post-deployment ‚Äúbootstrappers‚Äù to configure or initialize infrastructure resources._

---

## üîÑ When Do Provisioners Run?

| Phase         | Triggered When?                  |
| ------------- | -------------------------------- |
| `creation`    | After the resource is created    |
| `destruction` | Before the resource is destroyed |

You can control this behavior via `when = "create"` or `when = "destroy"`.

---

## üß≠ Provisioner Types

| Provisioner      | Where It Runs         | Example Use Case                           |
| ---------------- | --------------------- | ------------------------------------------ |
| ‚úÖ `local-exec`  | On your local machine | Log IP to local file, send Slack message   |
| ‚úÖ `remote-exec` | On the remote VM/EC2  | Install NGINX using SSH                    |
| ‚úÖ `file`        | Push file to remote   | Upload config files before installing apps |

## 1Ô∏è‚É£ `local-exec` ‚Äî Run Locally

Runs a command **on the Terraform host** (where you execute `terraform apply`).

### ‚úÖ Example: Append public IP to local file

```ini

  ami           = "ami-123456"
  instance_type = "t2.micro"

  provisioner "local-exec" {
    command = "echo ${self.public_ip} >> ips.txt"
  }
}
```

üß† `self` refers to the resource.

### ‚úÖ Example: Install NGINX via SSH

```ini
resource "aws_instance" "web" {
  ami           = "ami-123456"
  instance_type = "t2.micro"

  provisioner "remote-exec" {
    inline = [
      "sudo apt update",
      "sudo apt install nginx -y"
    ]
  }

  connection {
    type        = "ssh"
    host        = self.public_ip
    user        = "ubuntu"
    private_key = file("~/.ssh/web.pem")
  }

  key_name               = aws_key_pair.web.id
  vpc_security_group_ids = [aws_security_group.allow_ssh.id]
}
```

## 3Ô∏è‚É£ `file` ‚Äî Upload File to Remote VM

Use this to **copy a local file** to the remote machine.

### ‚úÖ Example: Copy config file

```ini
resource "aws_instance" "web" {
  ami           = "ami-123456"
  instance_type = "t2.micro"

  provisioner "file" {
    source      = "config/app.conf"
    destination = "/etc/app/app.conf"
  }

  connection {
    type        = "ssh"
    host        = self.public_ip
    user        = "ubuntu"
    private_key = file("~/.ssh/web.pem")
  }
}
```

## üîÑ Provisioner Execution Behavior

### üîß `when` Meta-Argument

Use `when = "create"` or `when = "destroy"` to control when provisioners execute.

```ini
provisioner "local-exec" {
  command = "echo Destroying..."
  when    = "destroy"
}
```

---

### üß® `on_failure` Meta-Argument

Controls what Terraform does if the provisioner fails:

| Value      | Behavior                     |
| ---------- | ---------------------------- |
| `fail`     | Stop the apply               |
| `continue` | Ignore and continue applying |

```ini
provisioner "remote-exec" {
  inline     = ["exit 1"]
  on_failure = "continue"
}
```

## üõë Why You Should Avoid Provisioners in Production

| ‚ö† Problem                   | üí• Why It‚Äôs Bad                                               |
| --------------------------- | ------------------------------------------------------------- |
| ‚ùå Not Idempotent           | They don‚Äôt track state. Rerunning may break or duplicate work |
| ‚ùå Difficult to Debug       | No retry, no logs, no visibility                              |
| ‚ùå Breaks Terraform Model   | Imperative logic inside declarative language                  |
| ‚ùå Poor CI/CD Compatibility | SSH issues, timeout, broken key forwarding, etc.              |

---

## ‚úÖ Recommended Alternatives

| Alternative      | Why It‚Äôs Better                              | Example                                |
| ---------------- | -------------------------------------------- | -------------------------------------- |
| `<user_data>`    | Native to cloud, easy to bootstrap VMs       | Install apps during EC2 startup        |
| Custom AMIs      | Fast launch, pre-installed dependencies      | Use Packer to build pre-baked images   |
| Config Mgmt Tool | Ansible / Chef / Puppet = full configuration | Integrate in CI/CD post-Terraform step |

## üß™ Terraform Plan Behavior

- They don‚Äôt change infrastructure state
- They only run at **apply** time
