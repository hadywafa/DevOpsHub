## üìå **What is PartiQL?**

**PartiQL (pronounced ‚Äúparticle‚Äù)** is a SQL-compatible query language that allows you to interact with **Amazon DynamoDB** using **SQL-like syntax**. It provides a **simpler and more familiar way** to perform CRUD operations without needing to construct complex DynamoDB API requests.

üöÄ **Why use PartiQL?**  
‚úÖ **SQL-like Syntax**: Easy for those familiar with traditional SQL.  
‚úÖ **Supports CRUD Operations**: `SELECT`, `INSERT`, `UPDATE`, `DELETE`.  
‚úÖ **Works with Nested Data**: Supports querying JSON-like structures.  
‚úÖ **Consistent Performance**: Operates efficiently under DynamoDB constraints.  
‚úÖ **Eliminates Complex API Calls**: No need for `GetItem`, `PutItem`, `UpdateItem`, etc.

## üìù **Basic PartiQL Query Syntax**

### ‚úÖ **1. Retrieve Data (`SELECT`)**

```sql

```

### ‚úÖ **2. Insert Data (`INSERT`)**

```sql

```

### ‚úÖ **3. Update Data (`UPDATE`)**

```sql

```

### ‚úÖ **4. Delete Data (`DELETE`)**

```sql

```
