## üîñ **1. Using `CreationPolicy` on an EC2 Instance**

‚úÖ **Best for:** Waiting for an **EC2 instance or AWS resource** to complete its bootstrapping before proceeding.  
‚úÖ **Does NOT need a `WaitCondition` or `WaitConditionHandle`.**  
‚úÖ **`cfn-signal` sends the success signal inside EC2‚Äôs UserData.**

### **How It Works**

1Ô∏è‚É£ CloudFormation **launches the EC2 instance**.  
2Ô∏è‚É£ EC2 **executes a setup script** (e.g., installs software).  
3Ô∏è‚É£ EC2 **sends a success signal using `cfn-signal`**.  
4Ô∏è‚É£ CloudFormation **waits for the signal** before moving forward.

### **CloudFormation Code**

```yaml

  MyEC2Instance:
    Type: AWS::EC2::Instance
    Properties:
      InstanceType: t2.micro
      ImageId: ami-12345678
      UserData:
        Fn::Base64: !Sub |
          #!/bin/bash
          yum install -y httpd
          /usr/bin/cfn-signal --success true --stack ${AWS::StackName} --resource MyEC2Instance --region ${AWS::Region}
    CreationPolicy:
      ResourceSignal:
        Count: 1
        Timeout: PT5M
```

‚úÖ **CloudFormation waits until `cfn-signal` is received before proceeding.**

üìå **Use `CreationPolicy` on EC2 when you only need to wait for that single instance to be ready!**

## üîñ **2. Using `WaitCondition` + `CreationPolicy` _(CreationPolicy on WaitCondition)_ (Multiple AWS Resources)**

‚úÖ **Best for:** Waiting for **multiple AWS resources** before proceeding.  
‚úÖ **Does NOT need a `WaitConditionHandle`.**  
‚úÖ **Multiple resources can send signals to a single `WaitCondition`.**

### **How It Works**

1Ô∏è‚É£ CloudFormation **creates an EC2 instance (`App1Server`)**.  
2Ô∏è‚É£ EC2 **runs a script and signals `App1ConfigWait` using `cfn-signal`**.  
3Ô∏è‚É£ CloudFormation **waits for the required number of signals** before proceeding.  
4Ô∏è‚É£ Once signals are received, **CloudFormation proceeds to create `App2Server`**.

### **CloudFormation Code**

```yaml
Resources:
  App1Server:
    Type: AWS::EC2::Instance
    Properties:
      InstanceType: t2.micro
      ImageId: ami-12345678
      UserData: !Base64
        Fn::Sub: |
          #!/bin/bash -xe
          yum install -y httpd
          /usr/bin/cfn-signal --success true --stack ${AWS::StackName} --resource App1ConfigWait --region ${AWS::Region}

  App1ConfigWait:
    Type: AWS::CloudFormation::WaitCondition
    DependsOn: App1Server
    CreationPolicy:
      ResourceSignal:
        Count: 1
        Timeout: PT5M

  App2Server:
    Type: AWS::EC2::Instance
    DependsOn: App1ConfigWait
    Properties:
      InstanceType: t2.micro
      ImageId: ami-12345678
```

‚úÖ **`App2Server` only starts after `App1Server` successfully completes setup.**

üìå **Use `CreationPolicy` on `WaitCondition` when waiting for multiple AWS resources!**

## üîñ **3. Using `WaitCondition` + `WaitConditionHandle` (Multiple External Resources)**

‚úÖ **Best for:** Waiting for **external scripts, APIs, or manual approvals** before continuing stack execution.  
‚úÖ **Requires a `WaitConditionHandle` to generate a pre-signed URL.**  
‚úÖ **External process must send an HTTP request to signal success/failure.**

### **How It Works**

1Ô∏è‚É£ CloudFormation **creates a `WaitConditionHandle`**, which generates a **pre-signed URL**.  
2Ô∏è‚É£ CloudFormation **starts a `WaitCondition`**, waiting for the signal.  
3Ô∏è‚É£ An **external process (script, API, human approval) sends a success signal** to the URL.  
4Ô∏è‚É£ CloudFormation **receives the signal** and resumes stack execution.

### **CloudFormation Code**

```yaml
Resources:
  MyWaitHandle:
    Type: AWS::CloudFormation::WaitConditionHandle

  MyWaitCondition:
    Type: AWS::CloudFormation::WaitCondition
    Properties:
      Handle: !Ref MyWaitHandle
      Timeout: "600"
      Count: 1
```

### **External Script to Send a Success Signal**

```sh
curl -X PUT -H 'Content-Type:' --data-binary \
    '{"Status": "SUCCESS", "Reason": "Setup Complete", "UniqueId": "MyExternalProcess", "Data": "Process Finished"}' \
    http://pre-signed-url-generated-by-MyWaitHandle
```

‚úÖ **CloudFormation pauses until the external process completes and sends a success signal.**

üìå **Use `WaitCondition` + `WaitConditionHandle` when waiting for an external process!**
