# **AWS Directory Service üåê**

AWS Directory Service is a **fully managed service** that allows organizations to run **Microsoft Active Directory (AD), Simple AD, or connect with on-premises AD** in the AWS Cloud.

It helps businesses:  
‚úÖ **Securely authenticate & manage users** across AWS and on-premises environments.  
‚úÖ **Enable Single Sign-On (SSO) & access control** for AWS resources.  
‚úÖ **Extend existing AD environments** into AWS with **AWS Managed Microsoft AD**.

üîπ **Why Use AWS Directory Service?**

- **Reduces IT overhead** ‚Äì Fully managed, no need for complex setups.
- **Seamlessly integrates with AWS services** ‚Äì EC2, RDS, WorkSpaces, FSx, and more.
- **Supports hybrid IT environments** ‚Äì Extend existing **on-premises AD** to AWS.

## üîë **Key Features of AWS Directory Service**

‚úî **Fully Managed Active Directory (AD)** ‚Äì No need to manage domain controllers manually.  
‚úî **Secure Authentication & Access Control** ‚Äì Supports **Kerberos, LDAP, NTLM, and MFA**.  
‚úî **Integration with AWS Services** ‚Äì EC2, S3, WorkSpaces, RDS, and FSx.  
‚úî **Scalability & High Availability** ‚Äì Automatically replicates AD data across multiple Availability Zones (AZs).  
‚úî **Hybrid AD Integration** ‚Äì Connects with **on-premises Active Directory** using **AWS AD Connector**.

## üõ†Ô∏è **AWS Managed Microsoft AD**

## üõ†Ô∏è **AWS AD Connector**

üí° **A lightweight directory service that acts as a proxy to on-premises AD.**

‚úî **Does not store user credentials in AWS** (No replication).  
‚úî Allows AWS applications **to authenticate with existing on-prem AD users**.  
‚úî Works well for **AWS Single Sign-On (SSO), Amazon WorkSpaces, and EC2 access control**.

üîπ **Use Case:**

- Organizations **already using on-prem AD** and want to **authenticate AWS workloads without full migration**.
- Businesses that need **AWS SSO for existing Active Directory users**.

## üõ†Ô∏è **Simple AD**

üí° **A lightweight, AWS-managed directory service for small-scale environments.**

‚úî **Built on Samba 4**, supports **basic LDAP, Kerberos, and DNS**.  
‚úî Cost-effective for **small businesses and simple authentication needs**.  
‚úî No **trust relationships** or **Group Policy support**.

üîπ **Use Case:**

- Startups & small businesses needing **basic authentication & user management**.
- Linux-based workloads that require an **LDAP directory service**.

## üèóÔ∏è **How to Set Up AWS Managed Microsoft AD**

### **1Ô∏è‚É£ Deploy AWS Managed AD**

1. Navigate to **AWS Directory Service Console** ‚Üí Click **Create Directory**.
2. Select **AWS Managed Microsoft AD**.
3. Choose a **fully qualified domain name (FQDN)** (e.g., `corp.example.com`).
4. Select **VPC & Subnets** for AD deployment.
5. AWS automatically **creates Domain Controllers across multiple Availability Zones (AZs)**.

---

### **2Ô∏è‚É£ Configure AWS Resources to Use AD**

‚úÖ **Join EC2 Windows Instances to AWS AD**

```powershell

Restart-Computer
```

‚úÖ **Enable AD Authentication for Amazon RDS**

```sql

```

‚úÖ **Apply IAM Role-Based Access Control**

```json
{
  "Effect": "Allow",
  "Action": "ec2:RunInstances",
  "Resource": "*",
  "Condition": { "StringEquals": { "aws:PrincipalTag/ADGroup": "Developers" } }
}
```
